1.配置在vscode中通过cmake编译

配置vscode基础的c++环境可以参照以下博客，这里给出的是在vscode中进行一系列的配置，最终可以在vscode中通过F7快速编译，编译生成的可执行文件在build文件夹下。ubuntu 上vscode使用cmake编译运行c++程序。但要注意这里面最后的debug部分内容不正确，需要进行额外操作。

2.CMakeLists.txt的编写

通过cmake进行编译需要在文件夹下编写CMakeLists.txt，此文件的基础语法可以参照如下博客CMakeLists详解。部分库的引用可以参考ROS包中的CMakeLists.txt的写法。注意相比于ROS包，部分头文件的写法会有一定区别，但对编译影响不大，主要是对代码高亮和引用有影响。

3.vscode中头文件引用不报错

vscode中头文件是基于其配置文件c_cpp_properties.json，与CMakeLists.txt无关，这使得要保持头文件引用不报错，需要配置c_cpp_properties.json。
按下F1快捷打开vscode的命令选项板，输入C/C++:编辑配置(JSON)，在文件夹下的.vscode文件夹中就会生成c_cpp_properties.json文件，在其中的”includePath”下添加头文件所在的目录。示例如下：

4.debug环境配置

debug可以简单理解为一种自动化的编译与运行过程，为实现debug，需要在vscode配置两个文件，一为tasks.json，一为launch.json文件。

tasks.json

tasks.json是配置自动化执行shell命令的文件，这里是配置debug过程中编译的命令。编译可以采用cmake、gcc、g++等，但需要配置好tasks.json中执行的命令。实际上配置tasks.json相似于通过命令行进行编译过程，下面是使用g++以及cmake进行编译的tasks.json文件的例子：

此处是基于g++进行编译，因此需要通过参数配置好编译中头文件连接、编译文件位置、生成可执行文件名称与位置等问题。即此处参数相当于cmakelists.txt的配置作用。基于g++的tasks.json可以自动生成，按下F1启动命令选项板，输入Tasks:Configure Default Test Task,选择C/C++:g++生成活动文件(cpp_test)即可生成模版，再在模版基础上修改即可。

以上为基于cmake编译，此处使用cmakelists.txt文件，主要在command设置上，上述例子中每次编译前删除上次编译生成的build文件，并重新进行编译。

launch.json

Launch.json的作用是告诉vscode调用DEBUG组件。launch.json的配置很简单，仅需要在模版基础上注意可执行文件的位置与名称即可。即如下参数的设置。

launch.json自动生成过程如下，点击左侧任务栏的第四个选项运行和调试，点击创建launch.json，在launch.json中点击添加配置，选择C/C++:(gdb)启动，然后更改program，使其可以读取的生成的可执行文件。
